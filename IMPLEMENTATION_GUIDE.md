&& sudo apt-get install -y terraform\n   \n   # Windows (using Chocolatey)\n   choco install terraform\n   \n   # Verify installation\n   terraform version\n   ```\n\n2. **AWS CLI** (>= 2.0)\n   ```bash\n   # macOS\n   brew install awscli\n   \n   # Ubuntu/Debian\n   curl \"https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip\" -o \"awscliv2.zip\"\n   unzip awscliv2.zip\n   sudo ./aws/install\n   \n   # Verify installation\n   aws --version\n   ```\n\n3. **kubectl** (Kubernetes CLI)\n   ```bash\n   # macOS\n   brew install kubectl\n   \n   # Ubuntu/Debian\n   curl -LO \"https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl\"\n   sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl\n   \n   # Verify installation\n   kubectl version --client\n   ```\n\n4. **jq** (JSON processor - optional but recommended)\n   ```bash\n   # macOS\n   brew install jq\n   \n   # Ubuntu/Debian\n   sudo apt-get install jq\n   ```\n\n### AWS Account Requirements\n\n- ✅ Active AWS account with billing enabled\n- ✅ IAM user with the following permissions:\n  - VPC management (create/read subnets, security groups)\n  - EKS (create/manage clusters and node groups)\n  - IAM (create roles, policies, OIDC providers)\n  - CloudWatch (create log groups)\n  - KMS (create/manage encryption keys)\n  - EC2 (launch instances, manage security groups)\n  - AutoScaling (create/manage ASGs)\n- ✅ VPC with:\n  - At least 2-3 private subnets in different availability zones\n  - NAT gateway for outbound traffic from private subnets\n  - Internet gateway for VPC\n  - Proper route tables configured\n\n### AWS Credentials Setup\n\n```bash\n# Configure AWS credentials\naws configure\n\n# You'll be prompted for:\n# AWS Access Key ID: [your-access-key]\n# AWS Secret Access Key: [your-secret-key]\n# Default region name: [us-east-1]  # Change as needed\n# Default output format: json\n\n# Verify credentials are working\naws sts get-caller-identity\n```\n\n---\n\n## Pre-Deployment Setup\n\n### Step 1: Gather VPC Information\n\nBefore starting deployment, collect the following information about your VPC:\n\n```bash\n# Get VPC ID\naws ec2 describe-vpcs --filters \"Name=tag:Name,Values=your-vpc-name\" --query 'Vpcs[0].VpcId' --output text\n\n# Get Private Subnet IDs (preferably from 3 different AZs for HA)\naws ec2 describe-subnets --filters \"Name=vpc-id,Values=vpc-xxxxxx\" \"Name=tag:Type,Values=Private\" --query 'Subnets[*].[SubnetId,AvailabilityZone]' --output table\n\n# Example output:\n# +-----------------+----------------+\n# | SubnetId        | AvailabilityZone|\n# +-----------------+----------------+\n# | subnet-12345678 | us-east-1a     |\n# | subnet-87654321 | us-east-1b     |\n# | subnet-11111111 | us-east-1c     |\n# +-----------------+----------------+\n```\n\n### Step 2: Verify VPC Configuration\n\n```bash\n# Check if subnets have NAT gateway route\naws ec2 describe-route-tables --filters \"Name=association.subnet-id,Values=subnet-12345678\" --query 'RouteTables[0].Routes' --output table\n\n# Check internet gateway\naws ec2 describe-internet-gateways --filters \"Name=attachment.vpc-id,Values=vpc-xxxxxx\" --output table\n```\n\n### Step 3: Clone/Update Repository\n\n```bash\n# Clone the repository (if not already cloned)\ngit clone https://github.com/sujithp28/terraform-aws-infrastructure.git\ncd terraform-aws-infrastructure\n\n# Switch to feature/eks branch\ngit checkout feature/eks\n\n# Verify you're on the correct branch\ngit branch\n# Should show: * feature/eks\n```\n\n---\n\n## Configuration Steps\n\n### Step 1: Create Terraform Variables File\n\n```bash\n# Navigate to examples directory\ncd examples/eks\n\n# Create terraform.tfvars from the example\ncp terraform.tfvars.example terraform.tfvars\n\n# Edit the file with your specific values\nvim terraform.tfvars\n# or\nnano terraform.tfvars\n# or\ncode terraform.tfvars  # if using VSCode\n```\n\n### Step 2: Update terraform.tfvars\n\nEdit `terraform.tfvars` with your AWS account details:\n\n```hcl\naws_region    = \"us-east-1\"              # Change to your region\nproject_name  = \"myproject\"              # Your project name\nenvironment   = \"production\"              # dev, staging, or production\n\n# IMPORTANT: Update these with your actual VPC and subnet IDs\nvpc_id             = \"vpc-0a1b2c3d4e5f6g7h\"   # Replace with your VPC ID\nprivate_subnet_ids = [\n  \"subnet-0a1b2c3d4e5f6g7h\",  # Replace with actual subnet IDs\n  \"subnet-1a1b2c3d4e5f6g7h\",  # At least 2-3 subnets recommended\n  \"subnet-2a1b2c3d4e5f6g7h\"\n]\n\nkubernetes_version = \"1.29\"              # Latest K8s version\n\n# System Node Group (for Kubernetes system pods)\nsystem_desired_size   = 3                # 3 for HA\nsystem_min_size       = 3\nsystem_max_size       = 10\nsystem_instance_types = [\"t3.medium\", \"t3a.medium\"]  # Cost-effective\n\n# Application Node Group (for your applications)\nenable_application_node_group = true\napplication_desired_size      = 3        # Start with 3\napplication_min_size          = 1        # Scale down to 1\napplication_max_size          = 20       # Scale up to 20\napplication_instance_types    = [\"t3.large\", \"t3a.large\"]  # Adjust based on workload\n\n# GPU Node Group (optional, for ML workloads)\nenable_gpu_node_group = false            # Set to true if needed\ngpu_desired_size      = 0\ngpu_min_size          = 0\ngpu_max_size          = 5\ngpu_instance_types    = [\"g4dn.xlarge\"] # GPU instances are expensive\n\n# API Endpoint Configuration\nendpoint_private_access           = true   # Keep cluster access private\nendpoint_public_access            = false  # Disable public access for production\npublic_access_cidrs               = []     # Empty when public_access is false\ncluster_endpoint_private_access_cidrs = [\"0.0.0.0/0\"]  # Restrict to your CIDR blocks\n\n# Logging Configuration\ncluster_enabled_log_types    = [\"api\", \"audit\", \"authenticator\", \"controllerManager\", \"scheduler\"]\ncluster_log_retention_in_days = 30        # Adjust based on compliance requirements\n\n# Add-ons\nenable_efs_csi_driver = false            # Set to true if using EFS\n\n# Tags for resource organization and billing\ntags = {\n  Environment = \"production\"\n  Team        = \"platform\"\n  CostCenter  = \"engineering\"\n  ManagedBy   = \"Terraform\"\n}\n```\n\n### Step 3: Review Backend Configuration (Optional but Recommended)\n\n```bash\n# For production, use remote state to avoid local state file\n# Edit backend.tf.example and uncomment the S3 backend configuration\n\ncp backend.tf.example backend.tf\nvim backend.tf\n\n# Update with your S3 bucket details:\n# - S3 bucket for state storage\n# - DynamoDB table for state locking\n```\n\n### Step 4: Initialize Terraform\n\n```bash\n# Initialize Terraform (downloads providers and sets up backend)\nterraform init\n\n# Expected output shows provider versions and backend initialization\n```\n\n---\n\n## Deployment Steps\n\n### Step 1: Validate Configuration\n\n```bash\n# Validate Terraform syntax\nterraform validate\n\n# Should output: Success! The configuration is valid.\n```\n\n### Step 2: Format Check (Best Practice)\n\n```bash\n# Check if code follows Terraform formatting standards\nterraform fmt -check -recursive .\n\n# Auto-format if needed\nterraform fmt -recursive .\n```\n\n### Step 3: Create Deployment Plan\n\n```bash\n# Generate plan to see what will be created\nterraform plan -out=tfplan\n\n# This will show:\n# - Number of resources to be created\n# - Resource details\n# - Expected time for deployment (15-20 minutes typically)\n\n# Save the plan to a file for review\nterraform plan -out=tfplan.json -json > plan.json\n```\n\n### Step 4: Review the Plan\n\n```bash\n# Read and understand the plan output\n# Check:\n# - Cluster configuration is correct\n# - Node groups are as expected\n# - Security groups are properly configured\n# - IAM roles are created\n# - Add-ons are enabled\n\n# Look for any unexpected changes\n```\n\n### Step 5: Apply Configuration\n\n```bash\n# Deploy the EKS cluster\nterraform apply tfplan\n\n# Or directly apply (will show plan first and ask for confirmation)\nterraform apply\n\n# Expected output:\n# - Cluster creation: ~10-15 minutes\n# - Node group creation: ~5-10 minutes\n# - Total time: ~20-30 minutes\n\n# Do NOT interrupt the process!\n```\n\n### Step 6: Get Cluster Outputs\n\n```bash\n# After successful deployment, retrieve important information\nterraform output -json > eks-outputs.json\n\n# Extract specific values\necho \"Cluster Endpoint: $(terraform output -raw eks_cluster_endpoint)\"\necho \"Cluster ID: $(terraform output -raw eks_cluster_id)\"\necho \"OIDC Provider ARN: $(terraform output -raw eks_oidc_provider_arn)\"\n```\n\n---\n\n## Post-Deployment Verification\n\n### Step 1: Configure kubectl\n\n```bash\n# Update kubeconfig to connect to the new cluster\naws eks update-kubeconfig --name $(terraform output -raw eks_cluster_id) --region us-east-1\n\n# Verify connection\nkubectl cluster-info\nkubectl get nodes\n\n# Expected output:\n# NAME                          STATUS   ROLES    VERSION\n# ip-10-0-1-123.ec2.internal   Ready    <none>   v1.29.x\n# ip-10-0-2-456.ec2.internal   Ready    <none>   v1.29.x\n# ip-10-0-3-789.ec2.internal   Ready    <none>   v1.29.x\n```\n\n### Step 2: Verify System Components\n\n```bash\n# Check system pods\nkubectl get pods -n kube-system\n\n# Verify add-ons are running\nkubectl get pods -n kube-system | grep -E 'coredns|aws-node|kube-proxy'\n\n# Check nodes are healthy\nkubectl describe nodes | grep -A 5 \"Conditions:\"\n\n# Verify CSI drivers\nkubectl get pods -n kube-system | grep csi\n```\n\n### Step 3: Check CloudWatch Logs\n\n```bash\n# View cluster logs in CloudWatch\nCLUSTER_NAME=$(terraform output -raw eks_cluster_id)\nLOG_GROUP=\"/aws/eks/${CLUSTER_NAME}/cluster\"\n\naws logs describe-log-streams --log-group-name \"$LOG_GROUP\" --query 'logStreams[*].logStreamName' --output table\n\n# View recent log events\naws logs tail \"$LOG_GROUP\" --follow\n```\n\n### Step 4: Test Kubernetes Functionality\n\n```bash\n# Deploy a test application\nkubectl create deployment nginx --image=nginx:latest\nkubectl expose deployment nginx --type=LoadBalancer --port=80 --target-port=80\n\n# Wait for load balancer\nkubectl get svc nginx -w\n\n# Test the application\nELB=$(kubectl get svc nginx -o jsonpath='{.status.loadBalancer.ingress[0].hostname}')\ncurl http://$ELB\n\n# Clean up test resources\nkubectl delete svc nginx\nkubectl delete deployment nginx\n```\n\n### Step 5: Verify Security Settings\n\n```bash\n# Check security groups\nSG_ID=$(terraform output -raw eks_cluster_security_group_id)\naws ec2 describe-security-groups --group-ids \"$SG_ID\" --output table\n\n# Check IAM roles\naws iam list-roles --query 'Roles[?contains(RoleName, `eks`)]' --output table\n\n# Verify KMS encryption\nKMS_KEY=$(terraform output -raw eks_kms_key_id)\naws kms describe-key --key-id \"$KMS_KEY\" --output table\n```\n\n---\n\n## Common Commands Reference\n\n### Terraform Commands\n\n```bash\n# Planning and Deployment\nterraform plan                    # Preview changes\nterraform apply                   # Deploy resources\nterraform destroy                 # Destroy all resources (use with caution!)\n\n# State Management\nterraform state list              # List all resources\nterraform state show resource     # Show specific resource details\nterraform refresh                 # Refresh state from AWS\n\n# Troubleshooting\nterraform plan -json              # Output plan as JSON\nterraform console                 # Interactive Terraform console\nTF_LOG=DEBUG terraform apply      # Enable debug logging\n```\n\n### kubectl Commands\n\n```bash\n# Cluster Information\nkubectl cluster-info              # Cluster endpoint and version\nkubectl get nodes                 # List all nodes\nkubectl describe node node-name   # Detailed node information\n\n# Pod Management\nkubectl get pods -A               # All pods in all namespaces\nkubectl get pods -n kube-system   # System pods\nkubectl logs pod-name -n namespace # View pod logs\nkubectl describe pod pod-name -n namespace  # Pod details\n\n# Deployment Management\nkubectl get deployments -A        # All deployments\nkubectl apply -f deployment.yaml  # Deploy from file\nkubectl delete -f deployment.yaml # Delete deployment\n```\n\n### AWS CLI Commands\n\n```bash\n# EKS Information\naws eks describe-cluster --name cluster-name  # Cluster details\naws eks list-clusters                          # List all clusters\naws eks list-nodegroups --cluster-name cluster-name  # List node groups\n\n# CloudWatch Logs\naws logs describe-log-groups                   # List log groups\naws logs tail log-group-name --follow         # Stream logs\n\n# Security Groups\naws ec2 describe-security-groups               # List security groups\naws ec2 describe-security-group-rules --filters Name=group-id,Values=sg-xxx\n```\n\n---\n\n## Troubleshooting Common Issues\n\n### Issue 1: Terraform Plan Fails with \"Access Denied\"\n\n**Solution:**\n```bash\n# Verify AWS credentials\naws sts get-caller-identity\n\n# Check IAM permissions\naws iam get-user --user-name your-username\n\n# Ensure credentials are loaded\necho $AWS_ACCESS_KEY_ID\necho $AWS_SECRET_ACCESS_KEY\necho $AWS_DEFAULT_REGION\n```\n\n### Issue 2: Nodes Not Joining Cluster\n\n**Solution:**\n```bash\n# Check security group rules\naws ec2 describe-security-groups --group-ids sg-xxx\n\n# Check IAM role\naws iam get-role --role-name eks-node-role-xxx\n\n# Check node group status\naws eks describe-nodegroup --cluster-name cluster-name --nodegroup-name nodegroup-name\n```\n\n### Issue 3: kubectl Cannot Connect\n\n**Solution:**\n```bash\n# Update kubeconfig\naws eks update-kubeconfig --name cluster-name --region region\n\n# Verify kubeconfig\ncat ~/.kube/config | grep -A 5 \"cluster-name\"\n\n# Check security group allows your IP\naws ec2 describe-security-groups --group-ids sg-xxx\n```\n\n### Issue 4: EBS/EFS CSI Driver Issues\n\n**Solution:**\n```bash\n# Check if driver pods are running\nkubectl get pods -n kube-system | grep csi\n\n# Check driver logs\nkubectl logs -n kube-system -l app=ebs-csi-controller\nkubectl logs -n kube-system -l app=efs-csi-controller\n\n# Verify IRSA role\naws iam get-role --role-name eks-ebs-csi-driver-role\n```\n\n---\n\n## Cleanup\n\n### Complete Cleanup\n\n```bash\n# WARNING: This will delete all resources!\n# Make sure you've backed up any important data\n\n# First, delete any resources created in the cluster\nkubectl delete all --all --all-namespaces\n\n# Then destroy Terraform resources\nterraform destroy\n\n# Verify all resources are deleted\naws eks list-clusters\naws ec2 describe-instances --filters \"Name=tag:eks:cluster-name,Values=cluster-name\"\n```\n\n### Partial Cleanup\n\n```bash\n# Remove only application resources\nkubectl delete deployments --all --all-namespaces\n\n# Or target specific resources\nkubectl delete ns my-namespace\n```\n\n---\n\n## Timeline Estimate\n\n| Task | Time
